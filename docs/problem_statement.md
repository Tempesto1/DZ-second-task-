# Спроектировать базу данных для веб-приложения ВУЗа

## Актуальность
Выбираем данную тему, так как без неё мы не сможем совершить банальный тест прототипа, для последующей доработки продукта

## Ограничения
Основным ограничением является закрытая база студентов/преподавателей всех ВУЗов

## Критерии успешности
1. Полностью рабочая и рандомнопроецируемая БД
2. Удачный fetch бд к веб-приложению без костылей

## Риски
потенциальных рисков нет
Непроверенное утверждение: приложение полностью автоматически составляет идеальное расписание без участия человека.
